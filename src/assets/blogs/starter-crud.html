What We're Building

Project Setup

Environment Setup

Before diving into Laravel, you'll need to set up your development environment:

  • Install XAMPP (for Apache, MySQL, and PHP)
  • Install Composer (PHP dependency manager)
  • Install Laravel via Composer

Creating the Laravel Project

Create a new Laravel project with this simple command:

composer create-project laravel/laravel crud-app

Database Configuration

Configure your database connection in the .env file:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel_crud
DB_USERNAME=root
DB_PASSWORD=

Code Walkthrough

1. Routes (web.php)

Routes define the URLs of your application and connect them to controller methods. Here's our CRUD routing setup:

Route::get('done', [DataController::class, 'index'])->name('data.index');
Route::post('done', [DataController::class, 'store']);
Route::get('view', [DataController::class, 'view'])->name('data.stored');
Route::get('destroy/{id}', [DataController::class, 'destroy'])->name('data.destroy');
Route::get('edit/{id}', [DataController::class, 'edit'])->name('data.edit');
Route::put('update/{id}', [DataController::class, 'update'])->name('data.update');

Key Points

  • Route::get handles GET requests (viewing pages)
  • Route::post handles form submissions
  • Route::put handles updates
  • Routes are named for easy reference in views

2. Controller (DataController.php)

Controllers handle the application logic. Our DataController manages all CRUD operations:

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Models\Data;

class DataController extends Controller
{
    // Show the form
    public function index() {
        return view('index');
    }

    // Store new data
    public function store(Request $request) {
        $data = $request->validate([
            'name' => 'required',
            'email' => 'required'
        ]);

        Data::create($data);
        return redirect()->route('data.stored')->with('success', 'Data created Successfully');
    }

    // View all data
    public function view() {
        $data = Data::all();
        return view('view', compact('data'));
    }

    // Edit existing data
    public function edit($id) {
        $data = Data::findOrFail($id);
        return view('edit', compact('data'));
    }

    // Update data
    public function update(Request $request, $id) {
        $validated = $request->validate([
            'name' => 'required',
            'email' => 'required|email'
        ]);

        $data = Data::findOrFail($id);
        $data->update($validated);
        return redirect()->route('data.stored')->with('success', 'data updated successfully');
    }

    // Delete data
    public function destroy($id) {
        $data = Data::find($id);
        $data->destroy($id);
        return redirect()->back();
    }
}

Controller Breakdown

  • index(): Displays the creation form
  • store(): Validates and saves new data
  • view(): Shows all records
  • edit(): Displays edit form for a specific record
  • update(): Processes edit form submissions
  • destroy(): Deletes a record

3. Model (Data.php)

Models interact with the database. Our simple Data model:

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Factories\HasFactory;

class Data extends Model
{
    use HasFactory;

    protected $guarded = false;
}

Model Explanation

  • use HasFactory: Enables model factory support
  • protected $guarded = false: Allows mass assignment for all fields
  • Laravel automatically knows this model connects to the "data" table

<a href="{{ route('data.destroy', $values->id) }}">

</a>

View Features

  • Uses Blade templating engine
  • @csrf includes CSRF protection token
  • route() helper generates URLs from named routes
  • Font Awesome icons for visual appeal
  • Tailwind CSS for styling
