Building a Habit Tracker with Laravel: A Beginner's Guide

Introduction

In today's fast-paced world, developing good habits is more important than ever. Whether it's exercising regularly, reading daily, or practicing mindfulness, tracking our habits helps us stay accountable and see our progress over time. That's why building a habit tracker app is not only a great programming project but also a tool you can use in your personal development.

In this tutorial, we'll walk through creating a basic habit tracker using Laravel, one of the most popular PHP frameworks. This project is perfect for beginners who want to get hands-on experience with:

  • Laravel fundamentals

  • MVC (Model-View-Controller) architecture

  • Database migrations

  • Basic CRUD (Create, Read, Update, Delete) operations

  • Routing in Laravel

Fun Facts

Why Build a Habit Tracker?

  1. Practical Application: Unlike abstract exercises, you'll build something you can actually use

  2. Fundamental Concepts: Covers core web development concepts in a manageable project

  3. Scalability: You can easily add more features as you learn

  4. Portfolio Piece: Demonstrates real-world skills to potential employers

Prerequisites

Before we begin, make sure you have:

  • PHP (7.4 or higher) installed

  • Composer installed

  • Basic knowledge of PHP and HTML

  • Laravel installed (we'll cover this in setup)

Setting Up the Project

First, let's create a new Laravel project:

bash
 
composer create-project laravel/laravel habit-tracker
cd habit-tracker

Understanding the Code Structure

Our habit tracker has three main components:

  1. Model: Habit.php (represents our data structure)

  2. Controller: HabitController.php (handles business logic)

  3. Views: habit-index.blade.php and habit-submit.blade.php (user interface)

Step 1: Database Migration

First, we need to set up our database structure. Laravel uses migrations to manage database schemas. Here's our migration file:

php
 
// database/migrations/xxxx_xx_xx_create_habits_table.php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

class CreateHabitsTable extends Migration
{
    public function up(): void
    {
        Schema::create('habits', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->text('description')->nullable();
            $table->boolean('completed_today')->default(false);
            $table->decimal('completion_percentage')->default(0.00);
            $table->integer('current_streak')->default(0);
            $table->enum('frequency', ['daily', 'weekly', 'monthly']);
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('habits');
    }
}

Migration Explanation:

  • id(): Auto-incrementing primary key

  • name: The name of the habit (e.g., "Morning Run")

  • description: More details about the habit

  • completed_today: Tracks if the habit was done today

  • completion_percentage: Tracks overall completion rate

  • current_streak: How many consecutive days/weeks/months the habit has been maintained

  • frequency: How often the habit should be performed

  • timestamps(): Automatically adds created_at and updated_at columns

Run the migration with:

bash
 
php artisan migrate

Step 2: Creating the Model

Laravel will automatically create a model when we generate the migration, but let's look at it:

php
 
// app/Models/Habit.php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class Habit extends Model
{
    use HasFactory;

    protected $fillable = [
        'name', 
        'description',
        'completed_today',
        'completion_percentage',
        'current_streak',
        'frequency'
    ];
}

The $fillable array specifies which fields can be mass-assigned, important for security.

Step 3: The Habit Controller

Now, let's examine the controller that handles our application logic:

php
 
// app/Http/Controllers/HabitController.php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Models\Habit;

class HabitController extends Controller
{
    // Display all habits
    public function index() {
        $habits = Habit::all();
        return view('habit-index', compact('habits'));
    }

    // Show the form to create a new habit
    public function submit_view() {
        return view('habit-submit');
    }

    // Store a new habit
    public function submit(Request $request) {
        $data = $request->validate([
            'name' => 'required',
            'description' => 'required'
        ]);

        Habit::create($data);
        return redirect()->route('routine.index')->with('success', 'New Record Added Successfully!');
    }
}

Controller Methods Explained:

  1. index():

    • Fetches all habits from the database using Habit::all()

    • Passes them to the habit-index view

    • Uses Laravel's compact() function to pass data to the view

  2. submit_view():

    • Simply returns the view with the habit creation form

  3. submit():

    • Validates the incoming request data

    • Creates a new habit record in the database

    • Redirects back to the index page with a success message

Step 4: Setting Up Routes

Routes define how URLs map to controller methods. Here are our routes:

php
 
// routes/web.php

use App\Http\Controllers\HabitController;

Route::get('routine-index', [HabitController::class, 'index'])
       ->name('routine.index');

Route::get('new-habbit', [HabitController::class, 'submit_view'])
       ->name('routine.submit.view');

Route::post('new-habbit', [HabitController::class, 'submit'])
       ->name('routine.submit');

Route Explanation:

  1. GET /routine-index:

    • Maps to index() method

    • Shows all habits

    • Named routine.index for easy reference

  2. GET /new-habbit:

    • Shows the habit creation form

    • Named routine.submit.view

  3. POST /new-habbit:

    • Handles form submission

    • Processes data via submit() method

    • Named routine.submit
